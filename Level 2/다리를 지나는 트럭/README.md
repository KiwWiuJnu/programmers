# 📝 [Programmers / Level 2] 다리를 지나는 트럭

> **분류**: 큐(Queue), 시뮬레이션  
> **문제 링크**: [다리를 지나는 트럭](https://school.programmers.co.kr/learn/courses/30/lessons/42583)

## 1. 문제 파악 및 제한 조건
* **핵심 목표**: 트럭들이 정해진 순서대로 다리를 건널 때 걸리는 최소 시간 계산.
* **데이터 범위 및 제약 사항**:
  * `bridge_length` (다리 길이): 1 이상 10,000 이하.
  * `weight` (다리 최대 무게): 1 이상 10,000 이하.
  * `truck_weights` (트럭 수): 1 이상 10,000 이하.
  * 다리 위에는 최대 `bridge_length`대의 트럭만 동시에 존재할 수 있으며, 이들의 무게 합은 `weight` 이하여야 함.

---

## 2. 논리 전개 및 풀이 과정 (Approaches)

### 🚨 1차 시도 (원시적인 접근 및 한계)
* **초기 설계**:
  * 다리 위에 트럭이 들어오고 나가는 이벤트를 "시간 점프" 방식으로 구현하고자 함.
  * 큐에는 단순히 다리 위 트럭들의 무게만 순서대로 삽입.
  * 새로운 트럭이 다리 무게 한계 등으로 진입하지 못하면, 큐에서 트럭을 `pop`하고 그에 비례해 흐른 시간(`answer`)을 보정식(`bridge_length - cnt`)으로 대충 때우려 함.
* **한계점**:
  * 큐에 단순 무게만 저장하여 트럭들의 개별 진입/탈출 시점 정보가 소실됨.
  * 이로 인해 여러 트럭이 복합적인 시간차로 다리에 머물 때(예: `[1, 1, 1, 9]` 등) 시간 왜곡이 발생하고, 다리 대수 제한(`q.size()`) 제어가 꼬여 `Assertion` 에러 또는 시간 오차 발생.

### ✅ 2차 시도 (0-채우기 시뮬레이션 및 최종 통과)
* **개선된 논리**:
  * 다리 위의 물리적인 위치와 칸수를 그대로 큐로 모사하는 **0-채우기(0-padding)** 기법 도입.
  * 큐의 크기를 항시 `bridge_length`로 유지하고 빈 공간은 `0`으로 채워둠.
  * 매 루프를 정확하게 **1초** 단위로 고정하여 시뮬레이션 진행:
    1. 매 초마다 다리 맨 앞 원소를 탈출시킴 (`q.pop()`, 가용 무게 복원, 탈출 원소가 실제 트럭이면 대수 `cnt` 감소).
    2. 이번 차례 대기 트럭이 다리에 올라갈 수 있는지 조건 검사 (남은 가용 무게 충분하며, 다리 위 트럭 대수 `< bridge_length`).
    3. 진입 가능하면 트럭 삽입 후 대기 순번 `i` 증가.
    4. 진입 불가능하면 `0`을 밀어 넣어 트럭들을 1칸씩 전진시킴.
  * 모든 트럭이 진입을 완료하면, 마지막 트럭이 다리를 빠져나오는 시간(`bridge_length`)을 최종 시간에 일괄 합산.
* **결과**: 시간 흐름과 큐의 크기가 완벽하게 동기화되어 모든 테스트 케이스 통과.

## ⏱️ 3. 복잡도 분석 및 성능 비교
* **시간 복잡도**: $O(\text{answer})$
  * 트럭들이 다리를 모두 건너갈 때까지 매 초 루프가 돎.
  * 최악의 경우(모든 트럭이 다리 길이만큼 간격을 두고 건널 때) 약 $O(N \times L)$ 시간에 비례.
  * 제한 조건 하에서 약 $10^8$ 이하의 연산이므로 1초 내에 충분히 통과.
* **공간 복잡도**: $O(L)$
  * 큐 `q`에는 항상 다리 길이 `bridge_length` 만큼의 원소(트럭 무게 또는 0)가 적재됨.
  * 따라서 공간 복잡도는 다리 길이에 비례함.

---

## ⚠️ 4. 엣지 케이스 및 실수 (Mistakes & Edge Cases)
* **다리 대수 제한 누락**: 무게뿐만 아니라 다리 위 트럭 수(`cnt`)가 `bridge_length` 미만인 경우에만 새 트럭을 받아야 함.
* **트럭 탈출 시 카운터 차감 누락**: `q.pop()`을 수행할 때, 나가는 원소가 실제 트럭(`> 0`)인 경우에만 트럭 개수 `cnt`를 줄여야 다리 상태 제어가 무너지지 않음.

---

## 🧠 5. 핵심 디테일 및 딥다이브 (Key Takeaways)
* **0-채우기를 통한 물리 구조 모사**: 복잡한 이벤트 기반 수식 계산보다 큐에 더미 데이터(`0`)를 넣어 시간을 1초씩 흘려보내는 방식이 구현의 난이도와 버그 발생 가능성을 대폭 낮춰줌.

```cpp
#include <bits/stdc++.h>
using namespace std;

int solution(int bridge_length, int weight, vector<int> truck_weights) {
    int answer = 0;
    int cnt = 0;
    queue<int> q;
    int len = truck_weights.size();

    for (int i = 0; i < bridge_length; i++) {
        q.push(0);
    }

    int i = 0;

    while (i < len) {
        answer++;

        weight += q.front();
        if (q.front() > 0) cnt--;
        q.pop();

        if (weight >= truck_weights[i] && bridge_length > cnt) {
            q.push(truck_weights[i]);
            weight -= truck_weights[i];
            cnt++;
            i++;
        } else {
            q.push(0);
        }
    }

    answer += bridge_length;

    return answer;
}
```
