# 📝 [Programmers / Level 3] 베스트앨범

> **분류**: 해시(Hash), 정렬(Sort), 우선순위 큐(Priority Queue)  
> **문제 링크**: https://school.programmers.co.kr/learn/courses/30/lessons/42579

## 1. 문제 파악 및 제한 조건
* **핵심 목표**: 장르별로 가장 많이 재생된 노래를 최대 두 곡씩 모아 베스트 앨범을 출시하기.
* **수록 기준**:
  1. 속한 노래가 많이 재생된 장르 먼저 수록.
  2. 장르 내에서 많이 재생된 노래 먼저 수록.
  3. 장르 내에서 재생 횟수가 같으면 고유 번호가 낮은 노래 먼저 수록.
* **시간 복잡도 유추**: 전체 곡 개수 $N$이 최대 10,000개이므로, 이중 루프로 탐색하여 $O(N^2)$이 발생하면 시간 초과 위험이 있음. 그룹화 및 정렬을 통해 $O(N \log N)$ 수준으로 최적화해야 함.

## 2. 논리 전개 및 풀이 과정 (Approaches)

### 🚨 1차 시도 (접근 및 한계)
* **초기 설계**: 전체 장르 중에서 가장 많이 재생된 '상위 2개 장르'만 추출하고, 해당 장르 곡들만 우선순위 큐에 담아 처리하려고 구상함.
* **한계점**: 
  * **문제 조건 오해**: 특정 2개 장르가 아니라 주어지는 **모든 장르**에 대해 수록 기준에 따라 최대 2곡씩 뽑아야 함.
  * **동점자 정렬 누락**: 기본 `priority_queue`는 재생 횟수가 같을 때 고유 번호 오름차순 정렬 조건을 자동으로 보장하지 않음.
  * **이중 루프 우려**: 장르를 돌 때마다 전체 배열을 매번 다시 탐색하면 비효율적인 $O(N^2)$ 시간이 소요됨.

### ✅ 2차 시도 (최적화 및 최종 통과)
* **개선된 논리**: 
  * `unordered_map<string, vector<pair<int, int>>>`를 사용하여 장르(Key)별로 속한 곡들의 {재생 횟수, 고유 번호}(Value)를 한 번의 순회로 묶어줌.
  * 장르의 수록 순서는 총 재생 횟수를 담은 별도의 `priority_queue`를 통해 구함.
  * **음수 인덱스 트릭**: 고유 번호에 `-1`을 곱해 저장한 뒤 해당 장르의 벡터를 기본 `sort()`하면, $O(1)$로 `.back()`과 `.pop_back()`을 써서 '재생 횟수 내림차순 -> 고유 번호 오름차순'으로 손쉽게 곡을 꺼낼 수 있음.
* **결과**: 전체 배열을 재순회하지 않고, 각 장르 벡터만 정렬한 뒤 끝에서부터 값을 추출하여 효율성을 크게 높임.

## ⏱️ 3. 복잡도 분석 및 성능 비교
* **시간 복잡도**: **$O(N \log N)$**
  * 데이터 순회 및 해시맵 그룹화: $O(N)$
  * 장르별 최다 재생 곡 정렬: 최악의 경우(한 장르에 모든 곡이 몰릴 때) $O(N \log N)$ 소요.
* **공간 복잡도**: **$O(N)$**
  * 해시맵 2개와 우선순위 큐에 총 $N$개의 곡 정보가 분산되어 저장됨.
* **💡 벡터 생성 메모리 오버헤드 분석**: 
  * 장르마다 벡터를 계속 생성하면 메모리 손해가 클 것으로 우려했으나, 결국 모든 벡터에 저장되는 데이터의 총합은 $N$개로 일정함. 
  * 벡터 자체의 컨테이너 오버헤드는 수백 KB 수준으로 코딩 테스트 환경에서는 매우 미미하며, 탐색 시간을 대폭 줄이는 훌륭한 트레이드오프임.

## ⚠️ 4. 엣지 케이스 및 실수 (Mistakes & Edge Cases)
* **곡이 1개뿐인 장르 (Edge Case)**: 
  * 무조건 2개씩 `pop_back()` 하려다 에러(Index Out of Bounds)가 발생할 수 있음.
  * `!v.empty()` 조건이나 `v.size()`를 반드시 체크하여, 곡이 하나뿐이면 1곡만 수록하고 넘어가야 함.
* **재생 횟수 중복 시 조건 누락**: 
  * 동점자 발생 시 고유 번호가 낮은 순서로 정렬해야 하는 추가 기준을 놓치면 오답 처리됨.

## 🧠 5. 핵심 디테일 및 딥다이브 (Key Takeaways)

### 1) C++ `unordered_map`의 자동 초기화 특성
* `unordered_map`은 존재하지 않는 Key에 접근할 때 알아서 빈 Value(이 경우엔 빈 `vector`)를 기본 생성(Default Construction)해 줌.
* 따라서 키의 존재 여부를 일일이 `if`문으로 검사할 필요 없이 `map[key].push_back(...)` 한 줄로 안전하게 삽입할 수 있음.

### 2) 커스텀 정렬 트릭 vs 람다(Lambda) 정렬
* **음수 인덱스 트릭**: 고유 번호에 `-1`을 곱하여 C++ `sort`의 기본 동작(오름차순)을 역이용하는 것은 코딩 테스트 환경에서 빠르고 기발한 방법임.
* **사용자 정의 정렬**: 가독성과 유지보수성이 중요한 실무나 협업 시에는 다음과 같이 람다식을 활용한 직관적인 커스텀 정렬 코드를 사용하는 것이 좋음.
  ```cpp
  sort(v.begin(), v.end(), [](const pair<int, int>& a, const pair<int, int>& b) {
      if (a.first == b.first) return a.second < b.second; // 횟수 같으면 번호 오름차순
      return a.first > b.first; // 기본은 횟수 내림차순
  });
